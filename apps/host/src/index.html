This is the beginning of a framework exploration

React vs Angular vs Vue vs Qwik vs everyone

In this case study I will develop a small ToDo app over and over using various frameworks. I try to make the app challenging so that most of the features of a frameworks can be used. The focus is on developing this application, without any additional libraries.

Then I compare

  • Mental Model - What general thinking is required in order to understand the framework?
  • Structure/Flexibility - How opinionated and configurable is the framework?
  • State Management - How does the application share a state?
  • Features Out of Box - How much can be done without external libraries?
  • Performance - How big is the Bundle? How good will it perform during runtime?
  • Ease of Use - How is the learning curve from starter to master? (Biased towards React)
  • Level of Magic - How easy is it to understand what is going on?
  • Caveats - General things need to be aware of and other concerns!

What will not be compared

  • Server Side Rendering
  • Build Tools - There are too many alternatives
